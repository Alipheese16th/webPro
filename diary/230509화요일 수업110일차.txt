# 프로젝트 배포

jar는 순수 자바프로젝트
war는 웹자바프로젝트

우리는 웹프로젝트를 배포할것이기때문에 war로 익스포트해야함

war로 export하려고하면 jar파일들 라이브러리는 배포되지않는다
maven에 있는 라이브러리만 배포되는데

기존에 있는 자바라이브러리에 우리가 인위적으로 넣은 ojdbc6이 배포가 안되기때문에 
pom.xml에 ojdbc6 디펜던시를 추가해서 이것까지 배포하게 만들자

익스포트하면 war파일을 배포하는데

클라이언트라면 새로운 WAS(예를들어톰캣)에 구동해볼텐데

이걸 톰켓 서버 파일 내 webapps 안에 넣으면 컨텍스트루트에 자동 설정된다

톰캣을 실행해볼건데 그전에 포트바꿔야함 오라클이랑 충돌남 기본 8080이라

톰캣 conf파일의 server.xml 포트 수정

이제 진짜 톰캣 실행해보자 bin 폴더와서 cmd실행

startup  명령어 실행하면 톰캣이 구동된다  webapps 폴더 다시 확인해보면 bowow폴더가 생겨있고 자동적으로 컴파일이 되어있다


클라이언트 피씨에는

서버파일이랑 war파일만 있는거

우리가 배포한 war파일 가지고 서버에서 구동하는것이다

# node 작업시작

vscode 의 터미널에서 작업시작 
경로는 소스폴더 내 원하는 폴더

npm init   명령어 (pakage.json 생성)
npm install express   (express 라이브러리 설치)

node index.js  (index.js 실행)

npm install -g nodemon (소스 수정시 자동으로 컴파일해서 다시 서버시작하는 라이브러리   -g 옵션은 다른 프로젝트에서도 전역적으로 사용가능하게 설치)

이제부터 node 로 실행하는대신 nodemon으로 실행하면 되는데
권한문제때문에 에러날수도있음
nodemon index.js가 에러나면 powershell을 관리자권한으로 실행해서 executionpolicy 실행결과가 Restricted이면 허가한 거만 실행해줌. 그래서 set-executionpolicy unrestricted 실행(Y)

여기서도 css폴더를 사용하려면 스프링처럼 컨트롤러에서 요청경로를 찾기때문에 제외해줘야함

express.static() 으로 리소스용 폴더를 지정

뭔가 느낌이 컨트롤러에서 뷰페이지랑 작업처리하는게 스프링이랑 느낌이 비슷함

form에서 매개변수보내면 컨트롤러에서 request.body.파라미터명 이런식으로 접근해서 사용가능함

몽고디비 3.6 으로 npm 인스톨
